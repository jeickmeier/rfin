# Conventions

Standards and patterns that apply across the entire Finstack codebase. Following
these conventions ensures consistency between Rust, Python, and WASM layers.

- [Naming](naming.md) — `get_*` accessors, metric keys, module layout
- [Error Handling](error-handling.md) — `Result` types, `map_error`, binding errors
- [Testing](testing.md) — Unit, doctest, parity, integration, coverage gates
- [Documentation](documentation.md) — Standards, reference anchors, doc verification
