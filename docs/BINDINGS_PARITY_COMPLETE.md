## How to Use

### Migrating Code from Python to TypeScript

1. Read the [Migration Guide](book/src/bindings/migration-guide.md)
2. Use the [Naming Conventions](NAMING_CONVENTIONS.md) for quick conversions
3. Reference [API Comparison](book/src/bindings/api-reference.md) for specifics
4. Check [Side-by-Side Examples](book/src/bindings/examples.md) for patterns

### Migrating Code from TypeScript to Python

1. Read the [Migration Guide](book/src/bindings/migration-guide.md) (works both ways)
2. Reverse the [Naming Conventions](NAMING_CONVENTIONS.md) patterns
3. Reference [API Comparison](book/src/bindings/api-reference.md) for specifics
4. Check [Side-by-Side Examples](book/src/bindings/examples.md) for patterns

### Maintaining Parity

1. **Before committing changes:** Run `python scripts/compare_apis.py`
2. **Check parity report:** Review `PARITY_AUDIT.md` for gaps
3. **Run golden tests:** `pytest finstack-py/tests/test_parity_golden.py`
4. **CI will verify:** Automated checks on every PR

### Adding New Features

1. **Implement in both bindings** (or document gap in PARITY_AUDIT.md)
2. **Add golden value test case** if applicable
3. **Update API reference** if adding major new type
4. **Run parity scripts** to update reports
5. **CI will verify** naming conventions and parity
